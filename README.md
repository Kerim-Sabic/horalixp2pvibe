## Why Horalix P2P?

Most "sync" tools are just cloud storage in disguise вҖ” your files live on their servers, you pay monthly, and they throttle your speed. Horalix P2P is fundamentally different:

**Files travel directly from device to device, encrypted with TLS 1.3, over your network.**

No upload to a server. No middleman. No subscription. No account. Your data never leaves your control.

> *"The safest server is the one that never receives your files."*

## вңЁ Features

### Core Sync Engine
- **Block-level delta sync** вҖ” only the changed 128 KBвҖ“16 MB chunks transfer. Rename a 50 GB folder: **0 bytes** transferred
- **Parallel block fetching** вҖ” requests multiple blocks simultaneously from multiple peers
- **Real-time FS watching** вҖ” OS-native events propagate changes in seconds, not minutes
- **Four folder modes** вҖ” Send & Receive В· Send Only В· Receive Only В· Receive Encrypted
- **File versioning** вҖ” Trash Can / Simple / Staggered / External strategies

### Network & Connectivity
- **TCP + QUIC dual-stack** вҖ” QUIC multiplexes streams for dramatically higher throughput than raw TCP
- **Full NAT traversal** вҖ” UPnP, NAT-PMP, STUN punches through firewalls automatically
- **LAN auto-discovery** вҖ” finds devices on your network in milliseconds via mDNS, zero config
- **Global discovery** вҖ” connects devices across the internet, no static IP or port forwarding required
- **Relay fallback** вҖ” always connects, even behind the most restrictive networks

### Security
- **TLS 1.3** on every single connection вҖ” no exceptions, no plaintext
- **160-bit device IDs** вҖ” cryptographic fingerprints, impossible to forge or spoof
- **Encrypted folder sharing** вҖ” give untrusted nodes (VPS, cloud) an encrypted copy they can't read
- **CSRF protection** on all API endpoints
- **Zero central authority** вҖ” no server that can be subpoenaed for your data

### Modern Web UI
- **Deep dark theme** вҖ” navy + electric blue/cyan/green accent palette
- **Live stats bar** вҖ” real-time download/upload speed, file count, total size, uptime
- **Dual-channel speed graph** вҖ” rolling canvas chart, green download + blue upload lines
- **Transfer Mode card** вҖ” one-click mode switching with immediate REST API apply
- **Instant search** вҖ” filter folders and devices as you type
- **Keyboard shortcuts** вҖ” full navigation without touching the mouse

## рҹ“– Full Tutorial

### Step 1 вҖ” Launch

Run `horalix.exe` (Windows) or `./horalix` (Linux/macOS).

On first launch, Horalix P2P:
- Generates your **Device ID** вҖ” a unique 160-bit cryptographic fingerprint
- Creates a default `~/Sync` folder
- Opens the web UI at **http://127.0.0.1:8384**

The dashboard shows **Folders** on the left and **Devices** on the right.

---

### Step 2 вҖ” Add a Folder

1. Click **Add Folder**
2. Set a **Label**, **Path**, and **Type**:

| Type | Behavior |
|---|---|
| **Send & Receive** | Bidirectional вҖ” changes anywhere sync everywhere |
| **Send Only** | This device is the source of truth |
| **Receive Only** | Only receives, never pushes |
| **Receive Encrypted** | Stores ciphertext only вҖ” can't see contents |

---

### Step 3 вҖ” Connect Devices

**Device A:** Click **Actions вҶ’ Show ID** вҶ’ copy your Device ID  
**Device B:** Click **Add Remote Device** вҶ’ paste Device A's ID вҶ’ Save  
**Device A:** Approve the incoming connection notification

Both devices now find each other automatically вҖ” on LAN in milliseconds, over the internet via discovery.

---

### Step 4 вҖ” Share a Folder

Edit the folder вҶ’ **Sharing** tab вҶ’ check the device вҶ’ **Save**.  
The remote device gets a one-click accept prompt. Sync starts immediately.

---

### Step 5 вҖ” Pick Your Transfer Mode

Click a mode in the **Transfer Mode** card on the dashboard:

- **вҡЎ Turbo** вҖ” LAN, fast connections, urgent sync
- **вҡ–пёҸ Balanced** вҖ” everyday use, the right default
- **рҹҚғ Eco** вҖ” background sync, always-on servers
- **рҹ“ұ Mobile** вҖ” cellular, metered WiFi, battery-sensitive

The mode applies instantly via REST API. A toast confirms it.

---

### Step 6 вҖ” Enable File Versioning

Edit folder вҶ’ **File Versioning** вҶ’ choose:

| Strategy | When to use |
|---|---|
| **Trash Can** | Recover any deleted/overwritten file, any time |
| **Simple** | Keep last N versions |
| **Staggered** | Hourly вҶ’ daily вҶ’ weekly backups automatically |
| **External** | Run a script on every change |

---

### Step 7 вҖ” Reading Status Badges

| Color | Meaning |
|---|---|
| рҹҹў Up to Date | Fully synced |
| рҹ”ө Syncing | Transferring (shows % + bytes remaining) |
| рҹ”ө Scanning | Hashing local files |
| рҹҹЎ Paused | Sync paused by user |
| рҹ”ҙ Out of Sync | Remote has changes you haven't received |
| рҹ”ҙ Failed Items | One or more files couldn't sync вҖ” click to see which |

## рҹ”Ң REST API

Full programmatic control вҖ” use from scripts, home automation, CI/CD.

**Auth:** `X-API-Key` header. Find your key under **Actions вҶ’ Settings вҶ’ GUI**.

```bash
# Status
curl http://127.0.0.1:8384/rest/system/status -H "X-API-Key: YOUR_KEY"

# Apply Turbo mode instantly
curl -X PATCH http://127.0.0.1:8384/rest/config \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"options":{"maxSendKbps":0,"maxRecvKbps":0}}'

# Trigger rescan
curl -X POST "http://127.0.0.1:8384/rest/db/scan?folder=photos" \
  -H "X-API-Key: YOUR_KEY"

# Live event stream (SSE)
curl "http://127.0.0.1:8384/rest/events?events=StateChanged,FolderSummary" \
  -H "X-API-Key: YOUR_KEY"
```

## рҹӣЎ Security Model

| Threat | How we handle it |
|---|---|
| Eavesdropping | TLS 1.3 on every connection вҖ” non-negotiable |
| Man-in-the-middle | Device IDs are pinned to TLS certificates |
| Unauthorized device | Must be explicitly approved; IDs are 160-bit |
| Data corruption | SHA-256 verified before every block write |
| Web UI hijacking | CSRF token required on all state-changing requests |
| Relay compromise | Relay only forwards ciphertext вҖ” can't decrypt |

## рҹҗӣ Troubleshooting

| Symptom | Fix |
|---|---|
| Devices won't connect | Open port 22000 TCP+UDP in firewall, or enable Relay in settings |
| Stuck at X% | Click the item count вҖ” see which files are blocked |
| High CPU | Switch to рҹҚғ Eco mode, or raise `RescanIntervalS` |
| UI not reachable remotely | Set `STGUIADDRESS=0.0.0.0:8384` and configure a password |
| Windows Firewall blocks | Allow `horalix.exe` in Windows Defender Firewall |
| Database error on start | Delete `index-*.db` in config dir вҖ” they auto-rebuild |
| macOS "unidentified developer" | Right-click binary вҶ’ Open вҶ’ Open anyway |
| Out of sync after editing | Normal вҖ” wait a few seconds, or click Rescan |
